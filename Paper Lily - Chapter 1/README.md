# Paper Lily - Chapter 1

**[Versão em Português](./LEIAME.md)**

## Mod: Quicksave System (F5)
This mod allows you to save the game at any point during gameplay by pressing the **F5** key. The system automatically searches for an empty save slot, and if all slots are occupied, it overwrites the oldest save based on the file's modification date.

Made for version 1.1.6

---

## Files in this folder
For transparency and to facilitate future modifications, the following files are available directly in this folder:
*   `Player.cs`: The C# source code containing the injected F5 key logic.
*   `Lacie Engine.dll`: The original game engine DLL, already modified (can be used for manual installation).
*   `README.md`: This installation guide and documentation.

---

## Installation (User)
1.  Go to your game's root folder.
2.  **Important:** Back up your original `PaperLilyCh1.exe` file by renaming it to `PaperLilyCh1.exe.bak`.
3.  Download the modified `PaperLilyCh1.exe` file from the [Releases](https://github.com/FinuraBR/meus-mods-de-jogos/releases/tag/Paper-Lily-Chapter-1) tab.
4.  Place the downloaded executable in the game's root folder.
5.  Start the game as usual.

#### How to use
*   Press **F5** at any time while controlling the character.
*   A save icon will appear in the corner of the screen.
*   The mod will save to the game's standard slots (`slot1`, `slot2`, etc.).

## Technical Documentation (For Developers/Updates)

If the game receives official updates, the modified `PaperLilyCh1.exe` may stop working. Below is the technical roadmap for recreating the mod:

#### 1. Tools Used
*   **Godot RE Tools:** For extracting the binary content of the executable.
*   **dnSpy:** For reverse engineering and injecting C# (IL) code into the engine DLL.

#### 2. Modification Process
The code was injected into the game's main DLL: `Lacie Engine.dll`.

1.  **Location:** The original file resides internally in the executable or within the `.pck` package under `.mono/assemblies/Release/`.
2.  **Target Class:** `LacieEngine.Nodes.Player`.
3.  **Target Method:** `_Input(InputEvent @event)`.
4.  **Logic:** Captures Scancode `16777248` (F5), checks for slots via `System.IO`, and calls the native function `LacieEngine.Core.GameState.Save`.

#### 3. Compilation Notes (Decompilation Bug Fixes)
When recompiling the `Player` class in dnSpy, it is necessary to correct errors generated by the tool:
*   **IPlayer Interface:** Add `public bool Exists { get { return Godot.Object.IsInstanceValid(this); } }`.
*   **Object Ambiguity:** Change `(Object)` references to `(Godot.Object)`.

#### 4. Injected Code (C#)
```csharp
if (@event is InputEventKey keyEvent && keyEvent.Pressed && !keyEvent.Echo && keyEvent.Scancode == 16777248U)
{
    string slotDestino = "slot1";
    int maxSlots = 30;
    bool achouVazio = false;

    for (int i = 1; i <= maxSlots; i++)
    {
        string nomeSlot = "slot" + i;
        if (!LacieEngine.Core.GameState.SaveExists(nomeSlot))
        {
            slotDestino = nomeSlot;
            achouVazio = true;
            break;
        }
    }

    if (!achouVazio)
    {
        try {
            System.IO.DirectoryInfo dir = new System.IO.DirectoryInfo(ProjectSettings.GlobalizePath("user://save/"));
            System.IO.FileInfo[] arquivos = dir.GetFiles("slot*.sav");
            if (arquivos.Length > 0)
            {
                System.IO.FileInfo maisAntigo = arquivos[0];
                foreach (System.IO.FileInfo arq in arquivos)
                    if (arq.LastWriteTime < maisAntigo.LastWriteTime) maisAntigo = arq;
                slotDestino = System.IO.Path.GetFileNameWithoutExtension(maisAntigo.Name);
            }
        } catch { slotDestino = "slot1"; }
    }
    LacieEngine.Core.GameState.Save(slotDestino, false);
    return;
}
```
